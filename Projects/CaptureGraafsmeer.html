Playtesting

An overview of user testing and feedback gathered from playtests conducted for CaptureGraafsmeer.

How do we test?

We conducted in-person user tests in Watergraafsmeer with both team members and external testers. The user tests involved walking the route in-game, beginning at the client’s office. Feedback was collected through tester comments and recorded on video, uploaded to YouTube as unlisted for reference.

All user tests were in Dutch, as both the testers and the development team are Dutch-speaking. The tests were conducted in Amsterdam, resulting in some background noise. There is no face-cam footage since the game takes over the camera permissions on the phone.

Important Stuff

  • Tests conducted in Dutch with local testers.
  • Audio quality impacted by background noise from the test location.
  • No face-cam footage due to camera permissions in the game.

Tests

User Test 1

Full test footage: Watch here

  • "Oh, je moet draaien met je vinger?" – The user was confused about camera control. Fix: Added a section to the introduction explaining camera control.
  • "Ja, leuk" – The user enjoyed the concept. No action required.
  • "Het uitzoomen van de camera is wel irritant." – Camera zoom-out was frustrating. Fix: Expanded on-camera controls in the tutorial.
  • "Waar eindige we waaneer we klaar zijn?" – User was unclear about the route's endpoint. Fix: Added a bird's-eye map of the route in the introduction.
  • "Ding staat verkeerd" – Incorrect placement of "Battle of Watergraafsmeer." Fix: Corrected location.

User Test 2

Full test footage: Watch here

  • "Het is de enigste knop, daar moet het wel in zitten" – The user suggested the button could be clearer. Fix: Made the button more prominent in the introduction.
  • "Wanner ik een foto heb gemaakt kan ik nog op het gebouw drukken." – User could retake photos unintentionally. Fix: Limited photo interaction to one click per building.
  • "Boven of onder?" – Confusion about route direction due to map design. Fix: Provided clearer guidance in the map and route instructions.

Next Steps

  • Address usability issues by enhancing clarity and providing better guidance on navigation and interaction.
  • Investigate and resolve technical issues to ensure a smoother user experience and minimize crashes.
  • Consider implementing some user suggestions for future iterations, prioritizing those that align with the project's goals and constraints.


Overall, the user testing process provided valuable insights that will inform the refinement and development of the game, with the ultimate goal of delivering a more engaging and seamless experience for players.
